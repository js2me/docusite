# Версионированная документация

Docusite предоставляет селектор версий «из коробки» — анимированный выпадающий список в навигации, который позволяет переключаться между версиями документации.

## Настройка

Добавьте опцию `versions` в конфигурацию:

```ts
import { defineConfig } from 'docusite'

export default defineConfig({
  versions: {
    latest: '2.0.0',
    older: [
      { label: 'v1.x.x', link: '/v1/introduction/getting-started' },
    ],
  },
})
```

### Поля

| Поле | Тип | Описание |
|---|---|---|
| `latest` | `string` | Метка текущей версии (например, `'2.0.0'` или `'v3.0.0'`) |
| `older` | `DocusiteVersion[]` | Массив старых версий |

Каждая запись в `older`:

| Поле | Тип | Описание |
|---|---|---|
| `label` | `string` | Отображаемая метка (например, `'v1.x.x'`) |
| `link` | `string` | Ссылка на стартовую страницу версии (например, `'/v1/introduction/getting-started'`) |

## Как это работает

При включении версионирования docusite автоматически:

1. Добавляет компонент `NavVersionsFlyout` в навигацию
2. Определяет текущую версию по пути страницы (например, если путь начинается с `/v1/`, показывается метка v1)
3. Показывает метку `latest` для страниц текущей версии

## Структура директорий

Организуйте документацию по версиям в подпапках:

```
docs/
  index.md
  introduction/
    getting-started.md       ← текущая версия (latest)
  guide/
    configuration.md
  v1/
    introduction/
      getting-started.md     ← v1 документация
    api/
      overview.md
```

## Sidebar для версий

Настройте отдельный sidebar для каждой версии через префиксы пути:

```ts
sidebar: {
  '/v1/': [
    {
      text: 'Введение',
      items: [
        { text: 'Начало работы', link: '/v1/introduction/getting-started' },
      ],
    },
  ],
  '/': [
    {
      text: 'Введение',
      items: [
        { text: 'Начало работы', link: '/introduction/getting-started' },
      ],
    },
  ],
}
```

## Предупреждение в старых версиях

Рекомендуется добавлять баннер-предупреждение в markdown-файлах старых версий:

```md
> ⚠️ Вы просматриваете документацию для **v1**. [Перейти к последней версии →](/introduction/getting-started)
```
